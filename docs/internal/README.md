# 📋 INTERNAL DOCUMENTATION

> Development and strategic information for Kerus team members.

## What's in this folder?

This folder contains **internal documentation** used by the development team for planning, architecture decisions, and understanding the system's vision.

### Documents

| Document | Purpose | Audience |
|----------|---------|----------|
| **ARCHITECTURE.md** | Full-stack system design (Frontend React + Backend .NET 9) | Architects, Tech Leads, Backend devs |
| **OBJECTIVES.md** | Project vision, roadmap, personas, business model | Product managers, Executives, Team leads |
| **DOCUMENTACAO_FUNCIONAL.md** | App features, behaviors, user flows | Everyone (feature reference) |
| **ROADMAP_EXECUTIVO.md** | 16-week implementation timeline, KPIs, projections | Executives, Product managers |
| **PROJECT_REVIEW.md** | Project analysis: strengths, weaknesses, recommendations | Stakeholders, Tech leads |
| **DOCUMENTACAO_DECISOES.md** | Architectural and technical decisions (ADRs) | Backend devs, Architects |
| **DEPRECATIONS.md** | Deprecated docs and redirects | Everyone (navigation) |

## Who should read what?

### Product Managers / Stakeholders
1. Start with [OBJECTIVES.md](./OBJECTIVES.md) — vision & roadmap
2. Then [ROADMAP_EXECUTIVO.md](./ROADMAP_EXECUTIVO.md) — timeline & KPIs
3. Reference: [PROJECT_REVIEW.md](./PROJECT_REVIEW.md) — full analysis

### Developers
1. Start with [ARCHITECTURE.md](./ARCHITECTURE.md) — system design
2. Reference [DOCUMENTACAO_FUNCIONAL.md](./DOCUMENTACAO_FUNCIONAL.md) — feature specs
3. Check [DOCUMENTACAO_DECISOES.md](./DOCUMENTACAO_DECISOES.md) — decisions

### Designers
1. Start with [DOCUMENTACAO_FUNCIONAL.md](./DOCUMENTACAO_FUNCIONAL.md) — features
2. Go to [../../design/](../design/) for DESIGN_SYSTEM_MASTER.md

## Maintenance

These docs are updated regularly:
- **ARCHITECTURE.md**: When architecture changes
- **OBJECTIVES.md**: When roadmap changes (quarterly review)
- **ROADMAP_EXECUTIVO.md**: When timeline changes
- **DOCUMENTACAO_FUNCIONAL.md**: When features are added/removed
