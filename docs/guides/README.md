## Getting Started

### 👨‍💻 If you're building the backend
1. Read [../../internal/ARCHITECTURE.md](../internal/ARCHITECTURE.md) first (understand the overall design)
2. Follow [BACKEND_IMPLEMENTATION.md](./BACKEND_IMPLEMENTATION.md) step-by-step
3. Reference [../reference/API_CONTRACT.md](../reference/API_CONTRACT.md) for endpoint specs

### 🎨 If you're working on frontend
1. Read [../../CLAUDE.md](../../CLAUDE.md) (project context)
2. Reference [../design/DESIGN_SYSTEM_MASTER.md](../design/DESIGN_SYSTEM_MASTER.md) for design decisions
3. Use [GUIA_RAPIDO.md](./GUIA_RAPIDO.md) for common issues
4. When backend is ready, follow [FRONTEND_MIGRATION.md](./FRONTEND_MIGRATION.md)

### 🚀 For local development
1. Run [GUIA_RAPIDO.md](./GUIA_RAPIDO.md) — setup & troubleshooting
2. Check [../../CLAUDE.md](../../CLAUDE.md) for project structure
